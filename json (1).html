Historia

JSON es un formato más reciente, que surgió a principios de los años 2000. Fue creado como una alternativa más ligera y fácil de usar que XML, especialmente para intercambio de datos en aplicaciones web, y está basado en la sintaxis de los objetos de JavaScript.

Normas de Uso y Sintaxis

JSON utiliza una sintaxis más ligera y menos redundante, basada en pares de clave-valor y arrays. Por ejemplo:

  1. JSON debe estar escrito utilizando sintaxis de objetos y arrays.
  2. Las claves (keys) deben estar entre comillas dobles.
  3. Los valores de las claves pueden ser strings, números, booleanos, arrays, objetos o null.
  4. JSON no permite comentarios dentro del código.
  5. El uso de coma (,) es obligatorio entre cada par clave-valor, excepto para el último.
  6. Los strings deben estar entre comillas dobles.
  7. JSON es independiente del lenguaje de programación y se utiliza principalmente para intercambio de datos.
  8. JSON es más ligero y más fácil de analizar que XML.
  9. JSON no admite referencias circulares (estructuras recursivas).
        {
    "libro": {
        "titulo": "El Señor de los Anillos",
        "autor": "J.R.R. Tolkien",
        "fechaPublicacion": "1954"
            }
        }

Aplicaciones en el Ámbito Empresarial

  • APIs RESTful:

    JSON es el formato de elección para muchas APIs web, especialmente aquellas que siguen el estilo arquitectónico REST. Es fácil de usar en combinación con JavaScript, que es comúnmente utilizado en el desarrollo web.

  • Almacenamiento de datos:

    JSON se utiliza cada vez más como formato de almacenamiento de datos en bases de datos NoSQL, como MongoDB, debido a su flexibilidad y capacidad para representar datos complejos.

  • Ejempos

    1. Intercambio de Datos: JSON se utiliza para intercambiar datos entre un servidor y un cliente en aplicaciones web y móviles a través de APIs.
    2. Almacenamiento de Configuración: Muchas aplicaciones almacenan su configuración en archivos JSON debido a su legibilidad y facilidad de manejo.
    3. Persistencia de Datos: En muchas aplicaciones, JSON se utiliza para persistir datos en la base de datos, ya sea en su formato original o después de ser convertidos en un formato compatible con la base de datos.
    4. Transmisión de Datos en Tiempo Real: JSON se utiliza en aplicaciones de tiempo real como chats y sistemas de mensajería para transmitir mensajes entre el servidor y el cliente de manera eficiente.
    5. Configuración de Servicios Web: En servicios web, JSON se utiliza para definir la configuración de servicios, parámetros de solicitud y respuestas.
